摘要

作为一个Polkadot平行链，XP.network能够将NFT产品和服务部署到任何适用的区块链上，包括Facebook即将推出的Diem。 XP.network利用其开创的应用程序编辑器，让企业家、开发人员、产品制造商和相关项目可以使用无代码工具来创建NFT DApps，消除行业壁垒。

NFTs

NFTs NFT代币代表着独特的数字资产所有权。每个NFT都与其所有者的帐户相关联，存储在区块链中，安全性高。

NF（非同质化）代表着“不可替代”的物品或艺术品、收藏品、时尚、体育、游戏以及其他具有独特属性的品目。

  • 它们只存在于单个副本中，数量稀少。
  • 它们为名人所有或由名人制作。
  • 是有着市场需求的数字资产（音乐、视频、图像），仅为其所有者或所有者的继承人进行展示/转售其全部或部分。.

NFT中的“非”代表着可替代资源的对立面。同质化代币是相同的、互相可以替代的法律或金融术语。例如，任何一种法定货币（如美元、欧元、英镑等）的单价面额都能替代1美元，这里的1美元就是同质化代币。

NFT为资产所有者赚取更多利润

一些线上平台从订阅计划或广告中盈利，却从不支付内容提供者任何费用[1]。而其他一些从销售中盈利的平台则会收取60-85%的费用，只向作者支付15-40%的报酬[2]。此外，NFT区块链只收取交易费用。

NFT反盗版

在全球数字化的背景下，版权的概念变得越来越模糊。层出不穷的MP3或PSD副本常常与艺术家的原版资产难分真假。盗版数字资产会削减原版的合法销售，并对创作者及其销售网络造成重大损害。画家、摄影师、设计师或插画家不断沦为盗版的受害者。他们的作品在其不知晓的情况下被复制、出售或展示。盗版的追查繁琐拖沓，且并不总是能得到好的结果。盗版的衍生就像复制粘贴一样简单。NFT旨在将所有权存储在有着独特属性的代币中，并将其不可逆转地保存在分布式区块链中。这样的数字资产有着可追踪性，且始终与其原始创建者相关联。

NFT的兴起

NFT在去年收获了主流大受的青睐。Beeple以不低于69,000,000美元的价格在佳士得拍卖行售出一幅数字艺术拼贴作品 — 其拍卖价格名列拍卖史上第三名。SOCKS，数字化“袜子”代币，交易金额超过89,000美元；与此同时，Twitter的联合创始人兼首席执行官Jack Dorsey以2,900,000美元的价格出售了他的第一条推文。Mark Cuban、Gary Vaynerchuk、Lindsey Lohan、Pamela Andreson、3LAU、Shawn Mendes、The Kings of Leon、Grimes、 Paris Hilton，甚至连Snoop Dog都在纷纷议论（有些还创建了）NFT。今日，NBA与Dapper Labs合作，主要销售由NFT支持的NBA“精彩视频”。这些视频的市场需求激增，得到了数十万用户的争抢，而每个用户至少拥有一个NFT，使得某些视频的售价甚至高于25万美元。

Facebook推出的Diem

“全球有10亿人拥有手机，近5亿人拥有网络连接，但仍有17亿成年人无法访问金融系统，与传统银行隔绝。”[3] 

Diem区块链背后的理念，是为了创建嵌入Facebook产品中的全球线上点对点支付系统[4]。Facebook及其旗下产品已收获31.4亿活跃用户，包括Facebook、Instagram、WhatsApp 和Messenger[5]的线上用户。这意味着全球79亿人口[6] 中，有40%的人在使用Facebook及其可能包含Diem钱包的关联帐户。Diem协会也因此成为全球最具影响力的金融机构。

挑战

市场壁垒

NFT应用程序仅由开发人员或技术人员创建。随着NFT的流行，越来越多的非开发人员也想在市场分一杯羹，但创建NFT所需的工具和技能需要一般人付出大量的时间和资源。创建NFT驱动产品或服务所需的努力，往往让许多非技术型产品经理和企业家半途而废。另外，NFT解决方案的使用也需要用户了解加密资产的基本工作原理。这些限制条件不仅削弱了产品的创造力，也降低了NFT的应用。

能力有限

NFT应用程序仅适用于与其匹配的区块链。大多数应用程序无法实现与其他区块链的内置价值转移（不具备接收或发送数字资产到另一个区块链的能力）。另一方面，大多数项目也缺乏在不同链上部署姊妹产品的能力。市场上虽然有很多以NFT为导向的解决方案和产品，却鲜有NFT解析工具。 [b]

动态的生态系统

区块链技术仍处于起步阶段。软件的快速变化，导致DApp库和代码需要不断进行维护和更新，随之而来的是高昂的财务成本，尤其在DApp还未具备明显的收入流或商业模式时。

法规

虽然Diem协会已与中央银行[ banks[10]和金融机构合作，将Diem与现有的金融体系结合起来，但Diem的相关应用市场必然会受到严格监管。开发人员将很难在Diem线上环境中推行他们的应用程序。线上的“拥堵”，可能会让他们的申请搁浅几周甚至几个月。申请者的数量将远远超过批准者。

Apple 示例代表 — 苹果商店

我们就拿苹果商店举例。开发人员表示，每周有100,000个应用程序接受审查，其中有40%都会被拒绝[11]例如不符合安全性、源代码或UI设计准则。此外，应用程序的一个版本被批准，并不代表其后续版本也会被自动批准。每个版本都被视为一个新的应用程序，因此可以被多次拒绝[12].

解决方案

XP.network平台包括：

架构

1. 智能合约编辑器/Smart Contract Editor

一种无代码UI工具，允许用户在Move、Solidity和Rust中生成有效的可读代码，进而构建可在Diem、Polkadot、以太坊、BSC和其他区块链上运行的智能合约。 用户选择NFT用例、具体业务信息和区块链。 完成所有步骤后，流程如下：

  1. JSON请求被发送到服务器。.
  2. 经过清理和验证。
  3. 最后生成一个 JSON“意图”对象。

“意图”对象包含：

  1. 所选区块链的智能合约编程语言。
  2. NFT用例被转换为智能合约。
  3. 具体业务信息作为规则、函数和变量输入。

“意图”对象被发送到下一个平台工具 — 迁移模块。

迁移模块生成可读代码，可供用户在UI的代码选项卡中读取和编辑。

更有经验的用户得以阅读和编辑这些代码。代码选项卡窗口会侦听用户生成的事件。当用户编辑并保存新版本的代码时，后端和合规验证者会受到一个请求，来对先前生成的代码进行修改，并确保新代码没有错误。

由于用户编辑的代码可能有误或与某些区块链政策相悖，因此XP.network合规验证者将对代码进行测试。代码选项卡中包含验证面板，用来显示验证结果，确认代码无误且符合法规，或指出代码的错误并提供修改建议。

为了让XP.network实现与Polkadot pallet的交互，用户的应用程序必须注册一个帐户。为了确保过程的保密和连续，该步骤可从该工具的“Polkadot交互”选项卡中完成。 与该模块关联的所有数据将存储在数据库中，包括用户选项、错误代码以及智能合约编辑代码。

2. 迁移模块/MIGRATION MODULE

迁移模块包括：

  1. 预编程的审计代码模板，由“意图”对象的数据生成。
  2. 决策树负责构建所选NFT用例的结构。
  3. 可读代码生成器。
  4. 字节码编译器。

由于Move没有用来处理NFT的标准库，XP Network针对Move的NFT库是原始标准库的扩展。未来可能会添加其他库和协议。

3. 合规验证者/COMPLIANCE VALIDATOR

合规性验证者是代码的自动QA工具。同样受约于区块链的最新相关法规。

迁移模块的输出通常无误且符合法规。但由于结果代码可被手动修改，因此该工具可用来验证并报告错误。合规验证者的结果会在代码选项卡的“验证面板”中显示。

例如，如果用户尝试在Move的智能合约内复制资源（Diem政策明令禁止），那么提醒合规错误的窗口就会自动弹出。

4. 应用编辑器/APPLICATION EDITOR

应用程序编辑器可生成结果应用程序的UI和后端。

用户可以定义：

  1. 如何管理结果应用程序的用户和权限。
  2. 使用哪些主题和样式。
  3. 使用哪些组件。
  4. 如何在页面上放置组件。
  5. 将组件与智能合约功能相连接。

该模块可构建结果应用程序的前端、后端和数据库。自动QA工具将检查结果应用程序的功能。

结果应用程序将连接到XP.network Substrate pallet和选定的区块链上。快速安全且可达到搜索引擎优化。

5. POLKADOT VM HUB

VM Hub Layer是一个Substrate pallet，允许不同的虚拟机进行交互。Layer实现了数据格式的序列化和反序列化，并与其他VM共享存储，使其可以互相发送请求。例如，用户可以通过Move或Ink内的智能合约调用Solidity内的智能合约！反之亦然。

用户可以选择以下区块链：Avalanche、Binance、Cardano、Diem、Elrond、Ethereum、Heco、Polkadot 和 Solana。不同的区块链对应着不同的智能合约编程语言。例如，如果用户想实现与以太坊的交互，则Solidity将被选为目标语言。

该模块包括：

1. 字节码解串器 — 接收十六进制输入，并在汇编中返还可读代码。

2. 汇编代码转换器 — 创建以下键值对：

  • 已选智能合约编程语言。
  • 模板编号。
  • 用于填充智能合约的数据。

3. 字节码编译器 — 以前一阶段生成的键值对作为输入值，生成的智能合约语言字节码作为输出值。

4. Polkadot平行线程 — 通过XP.network协议，使用中继链回调机制与其他平行链和平行线程进行交流。由于现有机制是无连接和无状态的，因此无法自动发出消息或交易成功的通知。这就是为什么每笔交易都被唯一的令牌标记。在等待交易结果或识别与交易相关的传入消息时，令牌将与交易信息存储在一起。

一组预编程的审计代码模板已准备好被任意数据填充。收到请求后，模板将被传入数据填充，并立即编译为交易准备字节码。每个平台在一开始都有一组有限的准备代码模板。新模板会在之后定期加入进来。大多数用例将可适用于每个桥接平台。最后，原始智能合约字节码将被直接翻译成目标语言字节码，至此限制消除。

由于该架构是为可注入接口设计的，因此不仅限于使用Solidity、Move、Rust 和Ink！编程语言。其他语言也可以实施该适配器，只要其请求符合可适应接口。

6. XP.NETWORK测试网

运行Substrate平行链的小型计算机网络，能让我们的用户以零成本测试生成的应用程序，然后再连接到主网上。其他区块链的测试网也会在不久的将来推出。

7.XPNET功能令牌
PoS验证

PoS是一种为验证者池选择最佳验证者的机制。验证者持有的XPNET令牌越多，被验证者池指定的机会就越大。验证者可以获取提名人的份额支持。一个提名人一次最多可支持12个验证者。

这种机制可以锁定更多令牌，让系统更能抵抗接管攻击。基于最大权益总额机制的筛选将在验证者池中轮换权益，确保每个时代（24 小时）保留一个活跃验证者。只有活跃的验证者才能签署区块并获得交易费用。如果验证者对提名者不感兴趣，验证者会将佣金设置为 100%。这表示提名者将获得佣金的0%。

用户须谨慎选择验证者。如果一个活跃验证者离线，它会多次签署相同的区块（模棱两可）或出现其他错误。该验证者和提名者的权益可能会被削减（因违反协议而产生罚款）。离线会导致一小部分权益被削减，但最终区块的模棱两可或替代签署可能会导致权益的100%削减。

为避免系统接管被报告或中心化，模棱两可的节点数量与削减份额成正比。如果有超过 33％的节点为模棱两可，则100％的权益将被削减。Genesis将存在100个独立的验证者节点。

随着XP.network生态系统的发展，节点数量将逐渐增加到几千，来迎接新的合作和盈利机会。

交易

每笔交易都由独立的去中心化验证者进行仔细检查。内部和桥接中继验证者都能得到努力工作的褒奖。交易费用由提交交易的账户持有人支付。

通过削减权益来减少验证者的失误、降低响应时间、离线频率或恶意欺诈活动。交易费不仅为验证者提供收入，还确保了交易过程的安全可用。

如果交易费用不存在，交易网络会因没有限制而造成拥堵，或受到DDoS攻击。高昂的交易费用可以为合法交易释放网络空间。交易费用需在交易前支付。

XP.network计划分配高达总代币供应量的10%，通过补贴100%的额外交易费用来刺激验证者。

目前交易手续费的计算公式为：m * size(TX) + b，其中
m（乘数）等于0.0005 XPNET。
size(TX) 是交易字节码的权重。
b是设为0.001 XPNET的偏差。

交易的权重与执行和存储的复杂性相关。偏差可以防止泛洪攻击。 交易发起者可以添加一个tip，以在交易队列中获得更高的优先级，从而提高交易费用。交易费用的公式和费率未来将由代币持有者通过公投决定。

权益

提名者和验证者是XP.network中的两种主要权益制造者。提名者要么是非技术型XPNET 代币持有者，无法设置验证者节点，要么不具备足够的代币来成为验证者。提名者可以通过将其权益分配给信任的验证者来赚取部分交易费用。

验证者运行区块链的完整节点，并积极参与区块的签署和生产。验证者被授予交易费用，但对其以及提名者的权益负责。如果验证者出错或故意签署欺诈交易，验证者的声誉不保并且验证者和提名者的权益也会被自动削减。

权益削减通过撤回一定比例的权益，对验证者恶意、错误或不完整的工作进行惩罚。验证者应确保他们的节点不会离线。

跨链价值转移

得益于XP.network正在开发的众多跨链桥，代币持有者将能够在桥接区块链之间无缝转移其数字资产。跨链桥也是去中心化的。独立的中继验证者可以确保区块链之间资产的完整性，防止任何内容的丢失或复制。

XPNET代币将存在于Polkadot、Elrond、HECO、Cardano、DFINITY和许多其他区块链中。一旦帐户持有人提交将令牌传输到桥接区块链的请求，桥将锁定原始区块链智能合约中的请求金额，并广播该事件。

桥接中继验证者捕捉事件并签署交易。目标区块链中的智能合约采集验证者的签名。在收到第2/3*n+1个签名时，它就会将相同数量的包装令牌释放到指定帐户。

桥接区块链的代币能以同样的方式转移到XP.network。XPNET可用于买卖其他资产（例如 NFT）、借贷、储蓄或作为抵押品使用。

治理

XP.network将逐步引入去中心化的治理机制，允许XPNET代币持有者对进一步的网络发展做出执行决策。一旦理事会提出裁决提案，并从代币持有者中选取，公投就将开始。

一个复杂的权益加权投票系统，可以确保决策的透明度和公平性。代币越多或被锁定的时间越长，提供给选民的选票就越多。锁定期结束后，无论公投结果如何，锁定的代币都将归还到原始钱包。

如果理事会成员一致投票赞成变更提案，则将应用负面投票率偏差。这意味着，即使选民数量不到50%，提案的通过或拒绝仍需占多数（n% * 3 / 5 + 1）。其中n%是得票率。

如果理事会支持投票提案但未获得一致通过，则必须有超过51%的代币持有者投票“赞同”，提案才能被接受，以便保证整个社区的利益。如果在公投期间改进决定被接受，由理事会选举的技术委员会将负责上传更改代码。
